Cloud Drive RAG
Architecture Explained — All Technologies, Versions & Data Flow
Page 1 of 3 · Layman's Guide

What does this application do? It lets you upload documents (PDFs, Word files, spreadsheets) or connect your Google Drive, then ask questions about those documents in plain English. The app finds the most relevant paragraphs, passes them to an AI language model, and returns a cited answer — all in a few seconds.

Why "RAG"? RAG stands for Retrieval-Augmented Generation. Instead of asking the AI to remember everything (which causes hallucinations), we first retrieve the relevant text from your documents, then generate an answer based only on that text. This makes answers accurate and verifiable.

1
⌨️
You type a question
Your browser sends the question to the backend over HTTPS. A session ID tracks the conversation so the AI remembers previous messages.
React 18 · Axios 1.7
fetch() + SSE stream
2
🔐
Auth & Rate check
FastAPI checks your identity token (JWT). SlowAPI enforces a 30 requests/minute limit to prevent abuse.
python-jose 3.3
slowapi 0.1.9
3
✏️
Query is rewritten
The AI rewrites your question to be clearer and more specific. "What about storage?" becomes "What are the storage options in Google Cloud?"
Groq llama-3.3-70b
langchain-groq 0.2.4
4
🛡️
PII safety scan
The query is scanned for personal data (names, emails, SSNs, phone numbers). If found, it is blocked and never reaches the AI — protecting user privacy.
presidio-analyzer 2.2.354
spaCy 3.8.4 (12 MB model)
5
🔍
Vector search in documents
The question is converted to a list of 1,024 numbers (a "vector") by Cohere AI. Qdrant then finds the 5 most similar document chunks using cosine similarity math.
cohere 5.5.8 (embed-v3)
qdrant-client 1.9.1
6
📋
Context assembled
The 5 best document chunks are combined with your previous questions (from Redis). This "context window" is what the AI will read to answer you.
langchain 0.3+
redis 5.0.6
7
🤖
AI generates answer
Groq's llama-3.3-70b reads only the retrieved context and writes an answer with source citations. Words stream back to your browser token-by-token in real time.
Groq API · llama-3.3-70b-versatile
temperature=0 · max 2048 tokens
8
🔒
Output safety scan
The AI's response is scanned again for any PII that might have leaked from your documents. Any detected entities are replaced with <TYPE> before reaching your screen.
presidio-anonymizer 2.2.354
spaCy en_core_web_sm
9
✅
Answer + citations returned
Clean answer with source links appears in your browser. The turn is saved to Redis so the AI remembers this exchange in your next question.
sse-starlette 2.1.0
redis 5.0.6

Technologies Deep Dive
Every library, what it does, and why it was chosen
Page 2 of 3 · Tech Reference
Frontend — What Runs in Your Browser
⚛️
React
v18.3.1
The UI framework. Builds the Chat, Ingest, Metrics and Analytics tabs as interactive components. Updates only what changed — no full page reloads.
⚡
Vite
v5.3.1
Build tool and dev server. Compiles TypeScript + React into optimised files for the browser. Hot-reloads the page during development in under 100ms.
🔷
TypeScript
v5.4.5
Adds type safety to JavaScript. Catches bugs at compile time before the code runs. All API data shapes, component props and store types are defined.
🐻
Zustand
v4.5.2
Lightweight state manager. Holds chat messages, pipeline stage states, trace IDs and metric data. Components subscribe and re-render only when their slice changes.
🔑
Keycloak-JS
v24.0.5
Handles login via Keycloak identity server. Manages JWT tokens, refreshes them silently, and provides user email and roles (admin vs user) to the app.
🎨
Design System CSS
Custom · 700 lines
A hand-crafted global CSS file with colour tokens, spacing scales, button styles, card styles and animation classes. Every component inherits from it — no duplicate styles anywhere.
Backend — What Runs on the Server (Python 3.11)
🚀
FastAPI
v0.111.0
The web framework. Defines all API routes, handles HTTP requests asynchronously, auto-generates Swagger docs at /docs, and validates all incoming data via Pydantic.
🦄
Uvicorn
v0.29.0
The ASGI server that actually listens for HTTP connections and passes them to FastAPI. Uses Python asyncio, handles thousands of concurrent connections without blocking.
🔗
LangChain
v≥0.3.20
A toolkit for building LLM applications. Provides prompt templates, chain building, output parsers and integrations with 100+ AI services. Used for query rewriting and context assembly.
🕸️
LangGraph
v≥0.2.73
Builds the RAG pipeline as a directed graph of nodes. Each node is a step (rewrite, scan, retrieve…). Defines what happens after each step, including conditional routes (e.g. block if PII found).
⚡
Groq API
langchain-groq v0.2.4
Provides access to llama-3.3-70b-versatile — a 70-billion parameter AI model. Groq's hardware delivers responses 10× faster than most cloud LLMs. Used for both query rewriting and answer generation.
🧮
Cohere Embeddings
cohere v5.5.8
Converts text into 1,024-dimensional number vectors (embeddings). Used at ingestion time (for every document chunk) and at query time. The embed-english-v3.0 model understands semantic meaning, not just keywords.
📦
Qdrant Cloud
qdrant-client v1.9.1
A vector database that stores all document chunks as 1,024-dim vectors. Answers "which chunks are most similar to this query?" using cosine similarity. Hosted on Qdrant Cloud — no local server needed.
🔴
Redis
redis v5.0.6
An in-memory database for session history. Remembers the last 20 turns of your conversation (TTL 1 hour). If Redis is unavailable, the app continues without history — it never crashes due to Redis.
🛡️
Microsoft Presidio
v2.2.354 + spaCy 3.8.4
Open-source PII detection by Microsoft. Uses spaCy's en_core_web_sm NLP model (12 MB) to find names, emails, SSNs, credit cards, IPs and phone numbers. Runs twice: on input and on output.

After parsing, every file goes through the same pipeline:
Parse text → PII scrub → Split into 512-token chunks (64 overlap) → Embed with Cohere → Store in Qdrant
Text Chunker: RecursiveCharacterTextSplitter
Library: langchain v0.3+  |  Chunk size: 512 tokens
Overlap: 64 tokens (so no sentence is cut in half between chunks)
Separators tried in order: paragraph → line → sentence → word

ChatWorkflow Tab — Real-Time Pipeline Observer
The ChatWorkflow tab is an observability console built into the app. When you send a question, it shows every pipeline step executing live — like watching an assembly line. Each node shows its status (pending → running → completed), timing, and the exact functions called inside it. This helps developers understand exactly what the AI is doing and debug problems.

Observability & Monitoring Stack
OpenTelemetry SDK v1.24.0
Distributed tracing across all HTTP requests. Sends trace spans to any OTLP-compatible collector (Grafana Tempo, Jaeger). Disabled when no collector is running.
Prometheus Client v0.20.0
Exports metrics at /metrics endpoint. Tracks: node_duration_seconds (per pipeline step), docs_ingested_total (success/error), chunks_created_total.
In-Process Tracers
metrics_tracker.py — query stats, latency, sources.
analytics_tracker.py — node timings, token costs, errors. Both thread-safe, zero external dependency.
Security Layers
🔑
JWT Authentication
python-jose 3.3.0
Every API request must carry a valid Keycloak JWT token. The backend verifies the RS256 signature without calling Keycloak on every request.
🛑
Rate Limiting
slowapi 0.1.9
Maximum 30 requests per minute per IP address on query endpoints. Prevents automated abuse and runaway API cost.
🕵️
Dual PII Scanning
presidio 2.2.354
Personal data is blocked on input (before AI sees it) AND scrubbed on output (before you see it). Two independent safety nets for every query.
